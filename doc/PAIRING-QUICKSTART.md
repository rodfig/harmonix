## Implementation Steps

### 1. Write and run the compile script ✅ Done

`scripts/compile-wines.py` reads all `producers-*.json`, flattens producer fields
into each wine, appends best retail price, writes `data/wine-profiles/wines-compiled.json`.

Output: 447 wines, 285 priced.

```bash
python scripts/compile-wines.py
```

**Rerun this script whenever source data changes** — any edit to `producers-*.json`,
`csv-results.json`, or `price-results.json`. Takes under a second.

### 2. Define dish profiles

Create `data/menus/<menu>/dish-profiles.json` (pairing annotations) and `data/menus/<menu>/menu.json` (presentation layer).
See `doc/partial_pairing_example.md` for the full dry run that already shows which
wines match each dish and why — use it as the basis.

### 3. Add scoring function to pairing-engine.js

Replace the static data load (`pairings-simple.json`) with:
- Load `wines-compiled.json`
- Load `dish-profiles.json`
- Implement scoring: weighted structural distance + rule bonuses
- Return top 2–3 wines with score + generated reason text

### 4. Add to HTML

**In `index.html` and `mobile.html`:**
```html
<!-- In <head> -->
<link rel="stylesheet" href="css/pairing-modal.css">

<!-- Before </body> -->
<div id="pairingModal" class="pairing-modal">
    <span class="pairing-modal-close">&times;</span>
    <div class="pairing-modal-content">
        <!-- Content inserted by JavaScript -->
    </div>
</div>
<script src="js/pairing-engine.js"></script>
```

**In `booklet-page-menu.html` and `booklet-page-menu2.html`:**
```html
<div class="course-name-pt">
    <span class="dish-link" onclick="openDishImage('...')">Dish Name</span>
    <span class="wine-link" onclick="openPairing('dish-id', 'Dish Name')">🍷</span>
</div>
```

### 5. Test

- Click each 🍷 icon (10 dishes)
- Verify modal open/close (×, ESC, click outside)
- Verify wines with no price show `—`
- Test mobile — full-screen modal, no swipe interference
